# HernexAI Agent Rules

- Read `docs/codex/repository-context.md` before major design work.
- Preserve Astro, TypeScript, Tailwind CSS, GSAP, and Lenis.
- Do not replace Astro with Next.js or convert the project into a React application.
- Do not invent clients, testimonials, awards, analytics, campaign results, or business statistics.
- Do not copy reference-site source code, text, assets, or exact layouts. Extract and reinterpret design principles.
- Avoid generic AI-generated website aesthetics, repetitive card grids, and randomly staggered sloppy cards.
- Use semantic HTML and accessible interactions.
- Support `prefers-reduced-motion`.
- Design mobile layouts intentionally rather than scaling desktop down.
- Run the production build and browser verification before reporting completion.
- Do not claim success when tests, browser checks, or builds are failing.

Before adding a new dependency or changing the architecture, check the repository context and the relevant project skill. Keep application changes separate from generated QA output and skill resources.
