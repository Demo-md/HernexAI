# HernexAI Design Principles

## Core Principles

- Build an authored visual system with a clear point of view, not a collection of fashionable components.
- Let hierarchy come from typography, spacing, composition, and surface changes before decoration.
- Give each section one job in the story and one dominant visual idea.
- Use asymmetry when it improves narrative clarity; keep primary copy and actions in stable layout flow.
- Use negative space as pacing, not as accidental emptiness.
- Make mobile a deliberate composition with fewer simultaneous objects and shorter cognitive paths.
- Make the first viewport useful before enhancement code loads.
- Use visual proof only when it is real and verifiable.

## Prohibited By Default

- Repetitive three-column card grids.
- Every service inside a rounded card or every section inside a bordered container.
- Randomly rotated, floating, scattered, or deliberately sloppy cards.
- Bento grids in more than one section.
- Excessive border radius, glassmorphism, or generic purple-blue AI gradients.
- Meaningless glowing orbs, random particles, generic robot imagery, generated corporate people, fake dashboards, and decorative charts without real data.
- Identical section structures, centre-aligned text everywhere, or meaningless icon collections.
- Animation without narrative purpose.

Allow no more than **one conventional card-grid section on the homepage**. Before a card is introduced, evaluate an editorial column, typographic index, timeline, sticky narrative, process rail, full-width media frame, split layout, numbered list, diagram, visual sequence, or horizontal story. Every major homepage section must have a distinct composition.
